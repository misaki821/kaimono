## 判定ロジックの絶対ルール（このアプリの肝・壊さないこと）

新しい店で買うとポイントは2種類増える:

1. **① 商品自体につくポイント**（通常1% + SPU + 買いまわり倍率ぶん）
2. **② 過去購入分への押し上げ効果**（店数+1により過去の全対象購入に+1%）

- **実質価格の判定は①だけで行う。②を①に混ぜない**（②はその店で何を買っても発生するため、混ぜると割高な商品が安く見えてしまう）
- ②は「店数を増やすこと自体の価値」として**別枠表示**する
- 判定3段階: ✅買い（①だけで最安値より安い）/ 🤔微妙（単体では割高だが②で元が取れる）/ ❌見送り（②を足しても割高）

## 計算の前提（勝手に変えないこと）

- ポイント計算は**税抜価格**ベース（2022年4月〜）。送料・ラッピング代は対象外
- ショップカウントは「1注文 **税込1,000円以上**」
- 買いまわりボーナス上限は通常7,000pt だが**回により変動**（5,000/10,000ptあり）→ 設定タブで変更できる設計を維持
- ボーナス分は期間限定ポイント（翌月15日頃付与、付与月の翌月末で失効）
- ふるさと納税は2025年10月から店数カウント・ポイントとも対象外
- 参考価格はスーパー(refSuper)とAmazon(refAmazon)の2つを持ち、**安い方**と比較する
- Amazon価格の自動取得は不可（スクレイピングブロック/API条件）。検索リンク+手動入力方式を維持する

## 楽天商品検索APIの仕様（2026年新仕様）

- エンドポイント: `https://openapi.rakuten.co.jp/ichibams/api/IchibaItem/Search/20220601`
- 認証は2キー必須: applicationId（UUID形式）+ accessKey（`pk_...`）。設定は `settings.rakutenAppId` / `settings.rakutenAccessKey`
- 旧 `app.rakuten.co.jp` は2026年5月に停止済み。**旧仕様に書き戻さないこと**
- JSONP不可、`fetch` を使う
- 「許可されたウェブサイト」制限があるため**ローカルファイルからは動かない**。API機能の動作確認は公開URL上で行う

### APIエラーのトラブルシューティング（2026-07-10の調査で確定）

エラー時のレスポンスは `{"errors": {"errorCode": ..., "errorMessage": "..."}}` 形式（旧形式の `error`/`error_description` ではない）。アプリはこのメッセージを画面に表示する。

| エラーメッセージ | 意味と対処 |
|---|---|
| `Invalid Access Key` | applicationId / accessKey が発行値と不一致。楽天ウェブサービスのアプリ情報ページからコピーし直して⚙設定に貼り直す（前後の空白に注意） |
| `REQUEST_CONTEXT_BODY_HTTP_REFERRER_MISSING` | Refererが届いていない。ローカルファイル(file://)から実行している（→公開サイトで実行する）か、ブラウザ設定/拡張機能がRefererを削っている |
| HTTP 429 / 503 | 連続アクセスの一時制限。10秒〜翌日待って**1回だけ**再試行。連打しない（QPS 1） |

注意点:
- 楽天側アプリの許可サイトは `misaki821.github.io` で登録済み・正しい（2026-07-10確認）
- **キー(設定)はブラウザのlocalStorageにサイト単位で保存される** → ローカル版と公開サイト版は別。公開サイト側の⚙設定に入れること
- fetchは `referrer: location.href, referrerPolicy: "unsafe-url"` を明示指定している（削らないこと）
- 検索成功が未確認の場合（2026-07-10時点）: 上記の修正を公開済みで、ユーザーが公開サイトから再試行するのを待っている状態。**成功が確認できたらこの行を削除すること**

## 修正後の動作確認チェックリスト

コードを修正したら、コミット前に必ず確認する:

1. `index.html` をブラウザで開き、**コンソールにエラーが出ていない**こと（F12 → Console）
2. 5つのタブ（📦ストック管理 / 🛒買い物リスト / 🤔買うべきか判定 / 📜購入履歴 / ⚙設定）が全て開けること
3. **localStorage の既存データが壊れないこと**。データ構造（保存するキーやプロパティ名）を変更する場合は、旧データからの移行処理を必ず入れる
4. 判定タブで適当な値を入れて、✅/🤔/❌ の判定と実質価格が表示されること
5. 修正した機能そのものを実際に操作して確認すること
