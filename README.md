# 楽天マラソン × 日用品ストック管理

日用品のストック(在庫)管理と、楽天お買い物マラソンでの「賢い買い方」をサポートするツールです。
`index.html` をダブルクリックしてブラウザで開くだけで使えます。インストール不要・データはブラウザ内に自動保存されます。

## できること

| タブ | 内容 |
|---|---|
| 📦 ストック管理 | 消耗品の在庫と月間使用量を登録 → 「残り○ヶ月」「買い時」を自動表示。**在庫は開くたびに経過日数分だけ自動で減ります** |
| 🛒 買い物リスト | 注文をショップごとに管理。1,000円未満の警告、店数・倍率・獲得ポイント・上限までの残りを自動計算。**完走プランナー**で店数を増やす価値と埋め方候補を提案 |
| 🤔 買うべきか判定 | 楽天価格とスーパー/Amazon価格を「実質価格」で比較して買い・見送りを判定。**内容量を入れると100mlあたり等の単価で比較**。**過去の底値**とも比較。**楽天アプリID設定で楽天内の最安値を自動検索** |
| 📜 購入履歴 | マラソン終了時に保存。前回価格・底値をストック一覧に表示。**ポイント付与・失効カレンダー**と**CSV書き出し(Excel用)** |
| ⚙ 設定 | 上限ポイント・SPU倍率・5と0のつく日・期間限定ポイントの価値・楽天アプリID |

## お買い物マラソンの仕組み(要点)

- **買いまわり**: 期間中に「1注文1,000円(税込)以上」買ったショップ数に応じて倍率アップ。最大10店舗で10倍(通常1% + ボーナス9%)。**倍率は期間中の全対象購入にさかのぼって適用**される
- ポイント計算は**税抜価格**ベース(2022年4月〜)。送料・ラッピング代は対象外
- 買いまわりボーナスの**上限は通常7,000pt**(回によって5,000pt/10,000ptあり) → 毎回、開催ページで確認して設定タブで更新すること
- ボーナス分は**期間限定ポイント**: 翌月15日頃付与、付与月の翌月末で失効
- **エントリー必須**(期間中なら購入後でもOK)
- ふるさと納税は2025年10月から店数カウント・ポイントとも対象外
- SPU・「5と0のつく日」は別枠計算で、それぞれ別の上限あり

## 「買うべきか判定」の考え方(このツールの肝)

新しい店で買い物をすると、ポイントは2種類増えます。

1. **その商品自体につくポイント**(通常1% + SPU + 買いまわり倍率ぶん)
2. **過去に買った分への +1%**(店数が増えたことによる"押し上げ効果")

②はその店で**何を買っても**発生するので、「この商品は本当に安いのか?」の判定に②を混ぜると、割高な商品でも安く見えてしまいます。
このツールは **①だけで実質価格を計算してスーパー/Amazon価格と比較**し、②は「店数を増やすこと自体の価値」として別枠で表示します。

- ✅ 買い: ①だけで比較しても最安値より安い
- 🤔 微妙: 商品単体では割高だが、店数を増やす目的なら押し上げ効果で元が取れる
- ❌ 見送り: 押し上げ効果を足しても割高

## Amazon価格について

Amazonの価格を自動で取得することは、技術的な制限(Amazon側が外部からの自動取得をブロックしている・公式APIは条件付き)のため対応していません。
代わりに、品名を入力すると **Amazon検索 / Keepa(価格推移) / 楽天の安い順検索** をワンクリックで開けるリンクが出るので、見た価格を手入力してください。ストック一覧の🔍からも開けます。

## 楽天の最安値 自動検索(無料)

楽天内の価格は公式API(外部プログラム向けの窓口)で自動取得できます。**2026年に新仕様へ移行済み**(旧APIは2026年5月13日停止)。設定手順:

1. [楽天ウェブサービス](https://webservice.rakuten.co.jp/) に楽天IDでログインし、新規アプリを作成
   - 応募タイプ: ウェブアプリケーション / APIアクセススコープ: 楽天市場APIのみ / QPS: 1
2. 発行された **アプリケーションID**(ハイフン入りの英数字、UUID形式)と **アクセスキー**(`pk_` で始まる)の2つをコピー
3. このアプリの ⚙設定タブに両方貼り付けて保存

以降、🤔判定タブで品名を入れて「🔍 楽天の最安を自動検索」を押すと、楽天内の最安値が安い順に表示され、クリックひとつで判定できます。

⚠ 新仕様には「許可されたウェブサイト」制限があり、ローカルファイル(ダブルクリックで開く方式)からのアクセスは拒否される場合があります。その場合はGitHub Pages等での公開(下記)で解決します。

## スマホで使うには(運用中)

GitHub Pagesで公開済み: **https://misaki821.github.io/kaimono/** (PCでもスマホでも同じURL)

アプリ本体を更新したときは、リポジトリ(kaimono)に新しい `index.html` を「+ → Upload files」でアップロードすると1〜2分で反映されます。

## PC⇔スマホのデータ共有(クラウド同期)

ヘッダーの「☁クラウドに保存」「☁クラウドから読込」で、GitHubのGist(非公開のメモ置き場)経由でデータを共有できます。

初回設定(各端末で1回だけ):
1. [トークン発行ページ](https://github.com/settings/tokens/new?scopes=gist&description=kaimono-sync)を開く(gist権限が選択済み)→ Expiration を「No expiration」→ Generate token → `ghp_...` をコピー(この時しか表示されません。PCとスマホで同じトークンを使うので、コピーした文字列は一時的にメモに控えておく)
2. アプリの⚙設定タブ「GitHubトークン」に貼り付けて保存

日々の使い方:
- 入力した端末で「☁クラウドに保存」→ もう一方の端末で「☁クラウドから読込」
- 読込は丸ごと上書きなので、「入力はどちらか一方の端末で」を意識すると安全

セキュリティ: データは非公開Gistに保存され、GitHubトークン自体はクラウドにアップロードされません(各端末内にのみ保存)。

## データのバックアップ

データはブラウザのlocalStorage(ブラウザ内の保存領域)にあります。ブラウザのデータ削除で消えることがあるため、ヘッダーの「データ書き出し」でJSONファイルをときどき保存しておくのがおすすめです。「データ読み込み」で復元できます。

## 毎回のマラソンの使い方(おすすめの流れ)

1. マラソン開催 → **公式ページでエントリー** + 上限ポイントを確認して⚙設定を更新
2. 📦ストック管理で「買い時」「そろそろ」の品を確認
3. 🤔判定タブで1品ずつ楽天価格を入れて判定 → 買うものは「買い物リストに追加」
4. 🛒買い物リストで1,000円未満の注文・上限到達をチェックしながら購入、買ったら「購入済み」に
5. マラソン終了 → 「🏁マラソン終了」で履歴に保存(在庫数への加算も聞かれます)
6. 翌月15日頃にポイント付与 → **期間限定ポイントの期限に注意!**
